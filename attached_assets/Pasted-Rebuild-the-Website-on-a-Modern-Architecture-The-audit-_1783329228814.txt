MOST IMPORTANT REQUIREMENT
Existing URLs MUST NOT CHANGE

My current website is already indexed on Google.

Most URLs currently end with .php.

Examples:

/goa-tour-package.php

/manali-tour-package.php

/dubai-tour-package.php

These URLs are permanent.

Even though the new platform will NOT use PHP, the new application must continue serving these exact URLs.

Do NOT:

Remove .php
Redirect .php URLs
Rename pages
Change slugs
Modify URL structure

The .php extension is now simply part of the public URL and should be preserved for all existing pages.

Future pages created through the CMS may use cleaner URLs, but every currently indexed URL must continue working exactly as it does today.
